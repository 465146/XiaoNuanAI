【项目简介】

小暖是一个基于 AI 的 CBT（认知行为疗法）心理陪伴 Web 应用。
面向大学生心理健康场景，提供对话疏导、情绪追踪、PHQ-9 抑郁筛查、
情绪日记生成、微信消息接入等功能。

技术栈：Python FastAPI + MySQL 8.0 + DeepSeek API + OpenClaw Gateway

三、核心功能模块

【用户认证】
  - JWT token 登录/注册
  - bcrypt 密码哈希
  - 可选的邮箱验证码（QQ SMTP 发送 6 位验证码）
  - 注册 POST /api/auth/register  登录 POST /api/auth/login

【AI 对话 — 智能路由】
  - 普通聊天 → 直连 DeepSeek API（快）
  - 音乐/技能请求 → 走 OpenClaw Gateway localhost:18789
  - SSE 流式输出 + Markdown 渲染
  - 对话历史持久化到 MySQL，刷新不丢失
  - 每次对话后自动评估情绪分数并记录

【情绪仪表盘】
  - 30 天情绪分数折线图（ECharts）
  - PHQ-9 测试历史柱状图
  - 周度文字报告（平均分、趋势、风险评估）
  - 今日情绪 / 本周平均 / 最新 PHQ-9 三指标卡片

【PHQ-9 抑郁筛查】
  - 9 道国际标准题目，随机顺序
  - 严重程度分级：无/轻度/中度/中重度/重度
  - 结果自动反馈到情绪追踪系统（daily_scores）
  - Q9（自伤念头）>0 时自动提供危机热线

【情绪日记】
  - 一键 AI 生成当日日记（基于对话内容）
  - 日记列表 + 详情查看
  - 下载功能（Markdown / 纯文本两种格式）
  - 按日期归档，独立存储

【微信连接 — 每用户独立 bot】
  这是整个项目的核心亮点。每个网站用户可以连接自己的独立微信 bot：

  流程图：
    网站用户登录 → 点"连接微信" → 获取专属二维码（ilink API）
    → 用户微信扫码 → 微信确认授权 → Bot 创建
    → 后台保存 bot 凭据（MySQL + Gateway 账号目录）
    → 启动独立消息轮询线程（长轮询 getupdates）
    → 用户发微信消息 → Python 后台接收 → AI 处理 → 自动回复

  技术细节：
    - 使用 WeChat ilink API (ilinkai.weixin.qq.com)
    - bot_type=3, iLink-App-Id=bot, iLink-App-ClientVersion=131335
    - 二维码编码微信 liteapp 链接 (liteapp.weixin.qq.com)
    - 扫码后长轮询 get_qrcode_status 等待确认
    - 确认后获得 bot_token + ilink_bot_id + ilink_user_id
    - 同时保存两份：MySQL wechat_bots 表 + Gateway 账号目录
    - wechat_bot.py 后台线程 35s 长轮询等待新消息
    - 收到消息后调 cbt_agent.chat_stream() 生成回复
    - 通过 ilink sendmessage API 发回微信

  多用户隔离：
    - 用户 A → Bot A (bot_token_A) → A 的对话历史 → A 的 AI 回复
    - 用户 B → Bot B (bot_token_B) → B 的对话历史 → B 的 AI 回复
    - 每个 bot 独立轮询、独立上下文、互不干扰

【用户偏好与个性化】
  - 每个用户独立的显示名称、目标、情绪触发点、应对方式
  - 自定义大模型 API Key / Base URL / Model（不填用系统默认）

  - 偏好存储：文件系统 user_data/{username}/preferences.json

【Gateway 配置要点】

  - 端口：18789（仅监听 loopback，不对外暴露）
  - 认证：token 模式，token = ${DEEPSEEK_API_KEY}
  - HTTP API：已开启 /v1/chat/completions 端点
    调用方式：POST http://127.0.0.1:18789/v1/chat/completions
    模型名：openclaw/cbt
  - 默认 Agent：cbt（CBT 心理陪伴）
  - Control UI：开启了 allowInsecureAuth（本地调试用）

【Gateway 在项目中的角色】

  1. 为网站提供增强模式 — 当用户消息包含音乐/技能关键词时，
     网站后端将请求转发给 Gateway，Gateway 调用 skills 处理
  2. 微信消息最初通过 Gateway weixin 插件收发，
     后来改为 Python 直连 ilink API 以实现每用户独立 bot
  3. Gateway 仍可独立运行，为终端用户提供微信 CBT bot 服务

八、关键技术决策

  1. 【为什么 Python 直连 ilink API 而不是全用 Gateway？】
     Gateway 的 weixin 插件设计为单 bot 模式。要实现"每用户独立 bot"，
     需要 Python 后端直接调用 ilink API 创建和管理多个 bot，
     每个 bot 独立轮询消息、独立 AI 对话上下文。

  2. 【为什么智能路由？】
     普通聊天 → 直连 DeepSeek（快、轻量）
     Skills（音乐等）→ 走 Gateway（Gateway 有网易云音乐 skills）
     这样既保证了日常对话的速度，又保留了 Gateway 的扩展能力。

  3. 【为什么文件存储偏好？】
     偏好数据不频繁变更，文件存储比 MySQL 更简单。
     每个用户一个 JSON 文件，路径 user_data/{username}/preferences.json。

  4. 【verify_codes 表 vs Redis】
     比赛项目规模下 MySQL 足够，不需要引入 Redis。


九、部署到 Railway

  railway.toml 已配置：
    - build command: uv sync
    - start command: bash start.sh（启动 Gateway + uvicorn）
    - 需要配置环境变量：DEEPSEEK_API_KEY
    - 需要外挂 MySQL 数据库（Railway MySQL 插件）
    - Gateway 不在 Railway 上运行（仅本地 Windows 环境支持）

  ⚠ 注意：ilink API 的微信 bot 功能依赖本地网络环境，
  在 Railway 云端可能无法正常调用 ilink API。
  建议微信功能仅在本机测试使用，生产部署仅提供 Web 聊天功能。
